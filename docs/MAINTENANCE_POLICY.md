# Wartungs- & Dokumentationsrichtlinie

## Ziel

Die Familienzentrale soll langfristig wartbar bleiben.

## Pflicht bei Änderungen

Bei jeder Änderung an:

- Architektur
- Backend
- Datenbank
- Deployment
- Docker
- Windows IoT Setup
- Authentifizierung
- API
- Synchronisierung

müssen die zugehörigen Dokumentationen aktualisiert werden.

## Betroffene Dokumente

- SETUP.md
- ARCHITECTURE.md
- KIOSK_MODE.md
- WINDOWS_10_IOT_INSTALLATION.md
- NEXT_STEPS.md
- AUTH_PLAN.md

## Deployment-Regel

Neue Features gelten erst als vollständig, wenn:

1. Code funktioniert
2. Dokumentation aktualisiert wurde
3. Windows-IoT-Betrieb berücksichtigt wurde
4. zentrale Displayoptimierung geprüft wurde

## Fokus des Projekts

Die Familienzentrale ist primär:

```text
Ein zentrales Familien-Wanddisplay
```

Smartphones und Tablets sind Ergänzungen.
